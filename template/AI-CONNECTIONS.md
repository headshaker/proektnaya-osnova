# Подключение коммерческих нейросетей

[← Начать здесь](START-HERE.md) | [Пакеты контекста](CONTEXT-WORKFLOW.md) | [Работа с вложениями](INGESTION-WORKFLOW.md)

«Проектная основа» не требует конкретной нейросети. Для обычной работы достаточно собрать `context.md` или `ai-package.md` и загрузить его в веб-интерфейс выбранного сервиса.

## Самый простой и безопасный вариант

1. Соберите пакет:

       pwsh ./scripts/build-ai-package.ps1 -Profile standard -Check

2. Откройте выбранную нейросеть в браузере.
3. Создайте отдельный проект или рабочее пространство с названием вашего проекта.
4. Загрузите `.project/context/ai-package.md`.
5. Добавьте постоянную инструкцию:

> Используй приложенный пакет как контекст проекта. Не изменяй действующие решения молча. Отделяй факты, расчёты, допущения и рекомендации. При нехватке данных формируй открытые вопросы. Любое предлагаемое изменение сначала показывай владельцу проекта.

6. После важной работы перенесите подтверждённый результат обратно в канонические файлы проекта.

## ChatGPT и OpenAI

### Без программирования

- Откройте ChatGPT: <https://chatgpt.com/>.
- Создайте отдельный Project или новый рабочий чат.
- Загрузите `ai-package.md` как файл проекта.
- Не полагайтесь на память чата как на единственное место хранения решений.

Официальная справка: <https://help.openai.com/>.

### Через API

- Создайте ключ в панели OpenAI: <https://platform.openai.com/api-keys>.
- Храните его только в менеджере секретов или переменной окружения:

       $env:OPENAI_API_KEY="ваш-ключ"

- Документация API: <https://platform.openai.com/docs/quickstart>.

## Claude и Anthropic

### Без программирования

- Откройте Claude: <https://claude.ai/>.
- Создайте отдельный Project, если эта возможность доступна вашему тарифу.
- Добавьте `ai-package.md` в материалы проекта или прикрепите его к новому чату.

Официальная справка: <https://support.anthropic.com/>.

### Через API

- Создайте ключ в Anthropic Console: <https://console.anthropic.com/settings/keys>.
- Переменная окружения:

       $env:ANTHROPIC_API_KEY="ваш-ключ"

- Документация API: <https://docs.anthropic.com/en/api/getting-started>.

## Gemini и Google

### Без программирования

- Откройте Gemini: <https://gemini.google.com/> или Google AI Studio: <https://aistudio.google.com/>.
- Создайте отдельный чат и загрузите `ai-package.md`.
- Для корпоративных материалов используйте тариф и учётную запись, одобренные вашей организацией.

### Через API

- Создайте ключ: <https://aistudio.google.com/apikey>.
- Переменная окружения:

       $env:GEMINI_API_KEY="ваш-ключ"

- Документация: <https://ai.google.dev/gemini-api/docs/api-key>.

## Microsoft Copilot и Azure OpenAI

### Microsoft Copilot

- Откройте Copilot: <https://copilot.microsoft.com/>.
- Используйте корпоративную учётную запись, если проект содержит служебную информацию.
- Прикрепите `ai-package.md` к запросу или разместите его в разрешённом хранилище Microsoft 365.

### Azure OpenAI

Azure OpenAI обычно настраивает администратор облака. Потребуются ресурс Azure, развёрнутая модель, endpoint и секрет или корпоративная идентификация.

Документация: <https://learn.microsoft.com/azure/ai-services/openai/>.

Не записывайте endpoint и ключи в Markdown-файлы проекта. Используйте Azure Key Vault, переменные окружения или секреты CI/CD.

## YandexGPT

- Веб-доступ и облачные сценарии настраиваются через Yandex Cloud.
- Быстрый старт: <https://yandex.cloud/ru/docs/foundation-models/quickstart/yandexgpt>.
- Для автоматизации используйте сервисный аккаунт и механизм авторизации, рекомендованный Yandex Cloud.

Не сохраняйте IAM-токены и авторизованные ключи в репозитории.

## GigaChat

- Документация для разработчиков: <https://developers.sber.ru/docs/ru/gigachat/api/overview>.
- Для API зарегистрируйте приложение в официальном кабинете разработчика.
- Секрет авторизации храните вне проекта.

## Что выбрать

- ChatGPT — универсальная работа с проектами, текстами, анализом и кодом.
- Claude — анализ длинных документов и подготовка структурированных материалов.
- Gemini — сценарии, связанные с экосистемой Google и мультимодальными файлами.
- Copilot или Azure OpenAI — организации, работающие в экосистеме Microsoft и Azure.
- YandexGPT или GigaChat — российская облачная инфраструктура и соответствующие корпоративные требования.

Выбор поставщика не меняет структуру проекта. Канонические файлы остаются главным источником состояния.

## Подключение через API: обязательные правила

1. Ключ создаётся отдельно для каждого сервиса и окружения.
2. Ключ не вставляется в код, Markdown, `.env`, который попадает в Git, или скриншоты.
3. Для GitHub Actions используются GitHub Secrets.
4. Для сервера используется менеджер секретов.
5. Отправляется только минимально необходимый пакет контекста.
6. Перед автоматической отправкой проверяются персональные данные, коммерческая тайна и договорные ограничения.

Рекомендуемые имена переменных:

    OPENAI_API_KEY
    ANTHROPIC_API_KEY
    GEMINI_API_KEY

Для Azure, Yandex Cloud и GigaChat следуйте актуальной официальной схеме авторизации: она зависит от способа развёртывания и может меняться.

## Проверка перед отправкой

Перед загрузкой откройте `ai-package.md` обычным текстовым редактором и убедитесь, что там нет:

- паролей и ключей;
- персональных данных, которые нельзя передавать поставщику;
- лишних договоров и приложений;
- сведений, не относящихся к задаче.

Автоматическая сборка ничего не отправляет в интернет. Передачу выполняет пользователь или отдельная интеграция после явного решения.
