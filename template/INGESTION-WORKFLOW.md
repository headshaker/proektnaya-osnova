# Автоматическая обработка файлов

[← Начать здесь](START-HERE.md) | [Источники](SOURCES.md) | [Пакеты контекста](CONTEXT-WORKFLOW.md)

> [!warning] Технический справочник
> Руководителю достаточно добавить материал и попросить ИИ его обработать. Установка конвертера, команды и диагностика ниже предназначены для агента или технического специалиста.

Подсистема преобразует локальные PDF, DOCX, PPTX, XLSX и другие файлы в Markdown с помощью Microsoft MarkItDown. Полный результат хранится локально, а в пакет для нейросети входят только подходящие фрагменты.

## Зачем это нужно

Большой договор или презентация могут занимать десятки тысяч токенов. Вместо передачи всего документа сборщик выбирает разделы по теме задачи и показывает фактическое сокращение в отчёте.

Исходный файл остаётся каноническим:

    _attachments → MarkItDown → .project/sources → выбранные фрагменты → ai-package.md

Файлы `.project/sources` являются производным кэшем и не попадают в Git.

## Установка

Требуется Python 3.10 или новее.

Для PDF, Word, PowerPoint, Excel и Outlook:

    python -m pip install "markitdown[pdf,docx,pptx,xlsx,xls,outlook]"

Текстовые `.md` и `.txt` обрабатываются без MarkItDown. Это позволяет проверять основной сценарий без дополнительной зависимости.

## Добавление документа

1. Поместите файл в `_attachments`, например:

       _attachments/contracts/agreement.docx

2. Добавьте строку в [`SOURCES.md`](./SOURCES.md). В колонке «Источник» укажите относительный путь:

       _attachments/contracts/agreement.docx

3. Запомните ID строки, например `S-014`.

4. Создайте кэш:

       pwsh ./scripts/ingest-sources.ps1 -SourceId S-014

Повторный запуск не преобразует неизменившийся файл заново.

## Сборка одного пакета для нейросети

    pwsh ./scripts/build-ai-package.ps1 `
      -Profile standard `
      -ExpandSource S-014 `
      -SourceQuery "штрафы, сроки, SLA" `
      -RefreshSources `
      -Check

Результаты:

- `.project/context/ai-package.md` — готовый пакет;
- `.project/context/ai-package-report.json` — объём, выбранные фрагменты и сокращение;
- `.project/sources/S-014/full.md` — полный локальный Markdown;
- `.project/sources/S-014/chunks.jsonl` — адресуемые фрагменты;
- `.project/sources/S-014/manifest.json` — SHA-256, версия конвертера и состояние кэша.

## Основные параметры

- `-ExpandSource S-014,S-015` — какие источники раскрыть.
- `-SourceQuery "стоимость","ответственность"` — какие фрагменты искать.
- `-SourceTokenBudget 6000` — сколько токенов отдать вложениям.
- `-RefreshSources` — обновить устаревший кэш.
- `-Profile compact|standard|extended` — общий объём пакета.
- `-Check` — завершиться ошибкой при пропуске или превышении бюджета.

Настройки форматов, размеров и бюджетов находятся в [`SOURCE-INGESTION.json`](./SOURCE-INGESTION.json).

## Безопасность

- обрабатываются только файлы внутри `_attachments`;
- URL из [`SOURCES.md`](./SOURCES.md) не загружаются;
- плагины MarkItDown выключены;
- облачный OCR и внешние анализаторы не вызываются;
- символьные ссылки и выход за пределы проекта запрещены;
- размер входа и результата ограничен;
- ZIP по умолчанию не разрешён;
- API-ключи не требуются.

Сканированный PDF без текстового слоя может дать пустой или неполный результат. Такой документ следует распознать отдельно в разрешённой организацией OCR-системе и сохранить результат как новый контролируемый источник.

## Как оценить экономию

В `ai-package-report.json` смотрите:

- `sourceFullEstimatedTokens` — полный объём преобразованных документов;
- `sourceIncludedTokens` — объём выбранных фрагментов;
- `sourceReductionPercent` — доля исключённого содержимого.

Большое сокращение не означает полноту ответа. Для юридической проверки договора может потребоваться полный документ или несколько отдельных запросов по разделам.
