# Обновление проектной основы

[← Главная](README.md) | [Ежедневная работа](DAILY-WORK.md) | [История проекта](CHANGELOG.md)

Обновление запускается из распакованного архива новой версии, а изменяемый проект передаётся через `-ProjectPath`. Канонические материалы проекта — бриф, решения, вопросы, источники, профильные документы и передача контекста — автоматически не заменяются.

## Подготовка

1. Зафиксируйте текущие изменения проекта в Git.
2. Скачайте ZIP и SHA-256 новой версии и проверьте контрольную сумму.
3. Распакуйте выпуск отдельно от обновляемого проекта.
4. Сначала постройте план без изменения файлов:

       pwsh ./scripts/update-project.ps1 -ProjectPath "../my-project"

5. Изучите операции и конфликты. Затем примените согласованный план:

       pwsh ./scripts/update-project.ps1 -ProjectPath "../my-project" -Apply

## Гарантии

- управляемый файл заменяется без дополнительного разрешения только тогда, когда его SHA-256 совпадает с официальной версией исходного выпуска;
- изменённый управляемый файл создаёт конфликт; `-ForceManagedFiles` допустим только после ручного сравнения;
- новые пользовательские документы не заменяют одноимённые существующие файлы;
- перед записью создаётся резервная копия в `.project/backups`;
- при ошибке проверки применённые операции откатываются;
- `TEMPLATE-VERSION` изменяется только в рамках применяемой транзакции.

## Старые проекты без маркера версии

Версии `0.1.0` и `0.1.1` могли не содержать `TEMPLATE-VERSION`. Для них версия не угадывается:

    pwsh ./scripts/update-project.ps1 -ProjectPath "../old-project" -FromVersion 0.1.1

Указывайте `-FromVersion` только после сверки происхождения проекта. Неподдерживаемая или противоречащая маркеру версия отклоняется.

## Совместимость реестров

Файл `REGISTRY-SCHEMA.json` описывает поддерживаемые строки решений, допущений, вопросов и источников. Проверка принимает исторические ID с любым количеством цифр (`D-1` и `D-001`) и форматы таблиц версий `0.1.x–0.4.x`, но отклоняет повторяющиеся ID и неизвестное число колонок.

    pwsh ./scripts/validate-registries.ps1

При переходе на `0.4.0` добавляются нейтральные профили и локальный сборщик контекста. Существующий `CONTEXT-PROFILES.json` считается пользовательской настройкой и автоматически не заменяется.

## Восстановление

После успешного обновления резервная копия сохраняется до ручного удаления. Отчёт `update-report.json` содержит исходную и целевую версии, перечень операций и результат. При автоматическом откате изучите отчёт и устраните причину до повторного запуска.
