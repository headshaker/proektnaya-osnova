# {{PROJECT_TITLE}}

> [!summary] Назначение
> Управляемая база знаний и контур работы руководителя проекта с ИИ-оператором. Руководитель задаёт цели и принимает решения; нейросеть анализирует проект, выполняет роли виртуальных специалистов и самостоятельно поддерживает файлы.

> [!tip] Впервые открыли шаблон?
> Начните с инструкции [«Начните здесь»](START-HERE.md). Она рассчитана на руководителя без опыта работы с Git, PowerShell, Markdown и агентными нейросетями.

## Как работает система

1. Руководитель ставит задачу естественным языком.
2. ИИ читает `AGENTS.md` и актуальное состояние проекта.
3. ИИ выбирает виртуальных специалистов и проводит профессиональный анализ.
4. ИИ самостоятельно изменяет канонические документы, решения, вопросы и `HANDOFF.md`.
5. ИИ запускает проверки и показывает управленческий итог.
6. Существенные решения и публикация подтверждаются руководителем.
7. Obsidian используется как удобный экран базы, GitHub — как история и механизм согласования изменений.

Подробности: [модель ИИ-оператора](AI-OPERATING-MODEL.md), [виртуальные специалисты](VIRTUAL-SPECIALISTS.md), [практика постановки задач](PROMPTING-GUIDE.md).

## Быстрый старт

Техническая настройка выполняется один раз. Требуется PowerShell 7 (`pwsh`). Для автоматической обработки PDF, Word, Excel и презентаций дополнительно используется Python 3.10+ и Microsoft MarkItDown.

Файлы `TEMPLATE-VERSION` и `TEMPLATE-STATE.json` фиксируют версию, схему реестров и историю обновления основы.

1. Скопируйте содержимое папки `template` в новый репозиторий.
2. Выполните `pwsh ./scripts/init-project.ps1 -Title "Название проекта" -Slug "project-slug"`.
3. Подключите агентную нейросеть по инструкции [AI-CONNECTIONS.md](AI-CONNECTIONS.md).
4. Откройте корень проекта как vault в Obsidian по инструкции [OBSIDIAN.md](OBSIDIAN.md).
5. Передайте агенту первую задачу из [START-HERE.md](START-HERE.md).
6. Зафиксируйте первый контрольный рубеж в [HANDOFF.md](HANDOFF.md).

## Что руководителю достаточно говорить нейросети

Пример обычной задачи:

> Подготовь проект к выбору варианта реализации. Сам изучи репозиторий, выбери нужных виртуальных специалистов, сравни варианты, обнови канонические файлы и `HANDOFF.md`, добавь недостающие вопросы и запусти проверки. Не меняй цель, границы, бюджет и обязательства без моего подтверждения. В конце покажи рекомендацию, риски, изменённые файлы и решения, которые нужны от меня.

Больше примеров находится в [PROMPTING-GUIDE.md](PROMPTING-GUIDE.md).

## Правила публикации

1. Канонические файлы изменяются до производных представлений.
2. ИИ выполняет `pwsh ./scripts/validate-vault.ps1`.
3. ИИ собирает и проверяет `PROJECT.md`.
4. ИИ готовит дайджест изменений.
5. Публикация и слияние выполняются только в пределах разрешения владельца.

По умолчанию существенные изменения готовятся в отдельной ветке и draft pull request.

Локальный пакет без вложений собирается командой `pwsh ./scripts/build-context.ps1 -Profile standard -Check`.
Единый пакет с фрагментами вложений собирается командой `pwsh ./scripts/build-ai-package.ps1 -Profile standard -ExpandSource S-002 -SourceQuery "тема задачи" -RefreshSources -Check`.
Обе команды работают локально и ничего не отправляют внешним API.
