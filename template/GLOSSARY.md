# Словарь простых терминов

[Пульт руководителя](HOME.md) | [Начните здесь](START-HERE.md) | [Главная](README.md)

Термины не нужно запоминать. Словарь нужен, чтобы быстро понять сообщение ИИ или подпись в GitHub и Obsidian.

## Ежедневная работа

| Термин | Простое объяснение |
|---|---|
| ИИ-оператор | Нейросеть, которая не только отвечает в чате, но и самостоятельно поддерживает материалы проекта в пределах данных ей полномочий. |
| Пульт руководителя | Главная страница с состоянием, вопросами, рисками и частыми действиями. |
| Краткое состояние | Автоматически подготовленный обзор проекта для руководителя: прогноз, препятствия и решения. Техническое имя файла — [`STATUS.md`](./STATUS.md). |
| Передача контекста | Запись о последних изменениях и следующем действии, позволяющая продолжить работу без чтения старого чата. Техническое имя — [`HANDOFF.md`](./HANDOFF.md). |
| Источник | Документ, ссылка или иной материал, подтверждающий сведения проекта. |
| Допущение | Временно принятое предположение, которое ещё нужно подтвердить. Это не факт и не решение. |
| Канонический документ | Основное место, где поддерживается конкретный смысл. Остальные обзоры ссылаются на него, а не создают противоречащие копии. |

## Git и GitHub

| Термин | Простое объяснение |
|---|---|
| Репозиторий | Папка проекта с историей изменений и возможностью восстановить предыдущие версии. |
| Git | Механизм, который сохраняет историю файлов проекта. В обычной работе им управляет ИИ или технический специалист. |
| GitHub | Сервис для хранения общей истории и согласования изменений. Проект может использовать и другой совместимый сервис. |
| Commit / коммит | Одна сохранённая версия с коротким пояснением, что изменилось. |
| Branch / ветка | Безопасная отдельная линия изменений. Она позволяет подготовить работу, не меняя сразу общую версию проекта. |
| Pull request / PR | Страница согласования подготовленных изменений перед их включением в общую версию. |
| Draft pull request | Черновая страница согласования: работа ещё не объявлена полностью готовой. |
| Diff | Точное сравнение: что добавлено, изменено и удалено. |
| Push | Отправка сохранённой версии в общее хранилище. |
| Merge / слияние | Включение согласованных изменений в основную версию проекта. |

## Документы и программы

| Термин | Простое объяснение |
|---|---|
| Markdown | Обычный текстовый формат документов. Его можно читать без специальной программы. |
| Obsidian | Необязательное приложение, которое показывает папку проекта как связанную базу знаний. |
| Vault | Папка проекта, открытая в Obsidian. Отдельной базы данных при этом не создаётся. |
| JSON | Технический формат настроек. Руководителю не рекомендуется редактировать его вручную. |
| PowerShell / `pwsh` | Средство выполнения технических команд на компьютере. Обычно команды запускает агент или специалист. |
| Скрипт | Небольшая программа, автоматизирующая настройку, проверку или сборку документов. |
| Кэш | Временный результат обработки, который ускоряет повторную работу. Его можно пересоздать из исходных материалов. |
| Контекст ИИ | Сведения проекта, переданные нейросети для выполнения конкретной задачи. |
| Токен | Техническая единица объёма текста для нейросети. Не путать с секретным токеном доступа. |

## Управление проектом

| Термин | Простое объяснение |
|---|---|
| Выгода | Измеримое полезное изменение, ради которого выполняется проект. |
| Контрольный рубеж | Проверка готовности к переходу на следующий этап с заранее понятными критериями. Это не просто дата. |
| Условие допуска | Требование, которое обязательно выполнить до действия, запуска или выпуска. |
| Допустимое отклонение | Заранее согласованная граница по сроку, стоимости или объёму, внутри которой команда может действовать без новой эскалации. |
| Блокирующее условие | Препятствие, без устранения которого продолжение или выпуск запрещены. |
| Риск | Возможное будущее событие, которое может повлиять на проект. |
| Проблема | Уже произошедшее событие, отрицательно влияющее на проект. |
| Зависимость | Результат или действие другой стороны, от которого зависит проект. |
| Эскалация | Передача вопроса человеку с достаточными полномочиями, когда команда или ИИ не могут решить его самостоятельно. |

Если встретился неизвестный термин, попросите ИИ: «Объясни это одной фразой без профессионального жаргона и скажи, требуется ли от меня действие».
