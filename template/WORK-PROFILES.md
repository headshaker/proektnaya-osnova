# Рабочие профили проекта

[← Главная](README.md) | [Ежедневная работа](DAILY-WORK.md) | [Управление и риски](docs/04-governance-risks.md)

Профиль задаёт глубину ведения базы, но не меняет формат файлов. Поэтому проект может усиливать управление постепенно без миграции данных. При старте выберите один профиль и зафиксируйте выбор в [DECISIONS.md](DECISIONS.md).

## Лёгкий

Подходит небольшой команде, прототипу или короткому исследованию с низкой ценой ошибки.

| Область | Минимум |
|---|---|
| Бриф | цель, границы и ближайший результат |
| Решения | только необратимые или дорогие выборы |
| Вопросы | P0 и P1 с владельцами |
| Источники | материалы, непосредственно влияющие на решение |
| Ритм | HANDOFF перед передачей работы; дайджест перед публикацией |
| Контроль | локальная проверка и актуальная единая книга |

## Основной

Профиль по умолчанию для продуктовой или инженерной команды с несколькими участниками.

| Область | Минимум |
|---|---|
| Бриф | измеримые цели, границы, заинтересованные стороны и ограничения |
| Решения | контекст, варианты, последствия, основание и условие пересмотра |
| Вопросы | P0–P2, владелец, следующий шаг, критерий закрытия и срок |
| Источники | тезис, применимость, дата проверки и повторная проверка |
| Ритм | обновление HANDOFF после значимой работы; ротация истории раз в неделю |
| Контроль | CI на двух ОС, бюджет контекста и согласованный дайджест |

## Регулируемый

Подходит проектам с правовыми, безопасностными, финансовыми или отраслевыми требованиями. Конкретные обязательства определяются применимыми нормами и владельцем соответствия.

| Область | Минимум |
|---|---|
| Полномочия | ответственный и утверждающий для каждого допуска и исключения |
| Решения | доказательство утверждения, версия входных данных и трассировка к требованиям |
| Вопросы | срок, владелец, риск просрочки и правило эскалации |
| Источники | издатель, версия, область применимости, дата доступа и срок актуальности |
| Риски | индикатор, мера, владелец, остаточный риск и решение о принятии |
| Ритм | контрольная точка перед каждым допуском; неизменяемый журнал выпусков |
| Контроль | независимая проверка, сохранение артефактов и формальное разрешение владельца |

Профиль не заменяет юридическую, безопасностную или отраслевую экспертизу. Требования конкретного проекта фиксируются в [управлении и рисках](docs/04-governance-risks.md) и действующих решениях.
