# История изменений «Проектной основы»

Все заметные изменения самого шаблона документируются в этом файле.

## 0.6.0 — 2026-07-16

### Добавлено

- модель ИИ-оператора проекта, работающего как личный ассистент, технический писатель и исполнитель изменений в репозитории;
- явное разделение полномочий руководителя и автономных действий нейросети;
- каталог виртуальных специалистов и протокол разрешения разногласий между ролями;
- руководство по промптингу с примерами задач руководителя в привязке к проектным файлам;
- расширенное руководство по Obsidian: workspace, ссылки, обратные ссылки, свойства, поиск, граф, Canvas и Daily Notes;
- агентные инструкции для OpenAI Codex, Claude Code, Gemini CLI и GitHub Copilot;
- единый контракт [`AGENTS.md`](./template/AGENTS.md), требующий сохранять результат в канонических файлах;
- тест состава и содержания агентных инструкций;
- проверяемое обновление проектов `0.1.0–0.5.0` до `0.6.0`.

### Изменено

- основной пользовательский сценарий перенесён с ручной загрузки контекста на постоянную работу агентной нейросети с репозиторием;
- [`START-HERE.md`](./template/START-HERE.md), [`DAILY-WORK.md`](./template/DAILY-WORK.md), [`AI-CONNECTIONS.md`](./template/AI-CONNECTIONS.md), [`OBSIDIAN.md`](./template/OBSIDIAN.md) и главная навигация переработаны для руководителя без технической подготовки;
- чатовый режим оставлен как запасной вариант для анализа, а не как основной способ ведения проекта.

### Безопасность

- изменение цели, границ, бюджета, обязательств, принятых рисков и действующих решений требует подтверждения руководителя;
- значимые изменения по умолчанию готовятся в отдельной ветке и draft pull request;
- инструкции требуют минимальных прав агента, проверки diff и запрета хранения секретов в проекте.

## 0.5.0 — 2026-07-16

### Добавлено

- понятная пошаговая инструкция [`START-HERE.md`](./template/START-HERE.md) для руководителей и специалистов без технической подготовки;
- руководство по подключению ChatGPT/OpenAI, Claude/Anthropic, Gemini, Microsoft Copilot/Azure OpenAI, YandexGPT и GigaChat;
- локальная обработка PDF, DOCX, PPTX, XLSX и других вложений через Microsoft MarkItDown;
- детерминированный кэш преобразованных источников с SHA-256, версией конвертера и адресуемыми фрагментами;
- выбор релевантных фрагментов по поисковым словам и отдельному токен-бюджету;
- сборщик единого `ai-package.md`, объединяющий канонический контекст и выбранные части вложений;
- отчёт о полном объёме источников, включённых токенах и фактическом сокращении;
- сквозной тест импорта, повторного использования кэша и обнаружения устаревшего исходника;
- проверяемое обновление проектов `0.1.0–0.4.0` до `0.5.0`.

### Безопасность

- конвертер принимает только локальные файлы внутри `_attachments`;
- URL, плагины MarkItDown, облачный OCR и внешние анализаторы не вызываются;
- производный кэш хранится только в игнорируемой папке `.project`;
- ZIP не входит в список разрешённых форматов;
- инструкции запрещают хранение API-ключей в Markdown, коде и Git.

## 0.4.0 — 2026-07-15

### Добавлено

- нейтральные к поставщику профили контекста `compact`, `standard` и `extended`;
- локальный сборщик пакета контекста с явным бюджетом и резервом ответа;
- адресная выборка решений, допущений, вопросов и источников по ID или поисковой фразе;
- машинно-читаемый отчёт о полноте документов, разделов передачи и запрошенных записей;
- сквозные проверки локального режима, отсутствующих записей, неполного [`HANDOFF.md`](./template/HANDOFF.md) и безопасных путей.

### Безопасность

- локальный режим включён по умолчанию и разрешает запись только в `.project/context`;
- сборщик не выполняет сетевых запросов и не передаёт содержимое внешним API;
- экспорт требует явного параметра и всё равно ограничен границами проекта.

## 0.3.0 — 2026-07-15

### Добавлено

- планируемое и транзакционное обновление существующих проектов из отдельного выпуска;
- исторические SHA-256 управляемых файлов для версий `0.1.0–0.2.0`;
- резервное копирование, отчёт операций и автоматический откат при ошибке проверки;
- явная схема совместимых форматов решений, допущений, вопросов и источников;
- состояние шаблона с исходной, предыдущей и текущей версиями;
- сценарии успешной миграции, конфликтов, отсутствующего маркера версии и отката.

### Безопасность

- канонические материалы проекта не заменяются миграцией;
- изменённые управляемые файлы требуют ручного разрешения через `-ForceManagedFiles`;
- путь каждой операции проверяется на выход из проекта и точки повторного анализа.

## 0.2.0 — 2026-07-15

### Добавлено

- безопасная команда добавления решений, вопросов и источника с автоматическими ID;
- автоматическая ротация устаревших записей [`HANDOFF.md`](./template/HANDOFF.md) в проектный changelog;
- проверка входящих и исходящих локальных ссылок для заметок в `docs` и `_inbox`;
- генератор игнорируемого дайджеста перед коммитом;
- лёгкий, основной и регулируемый профили ведения проекта;
- сквозные тесты ежедневного цикла, включая параллельное добавление записей.

### Изменено

- ежедневный протокол включён в единую книгу проекта;
- версия распространяемого шаблона повышена до `0.2.0`.

## 0.1.2 — 2026-07-15

### Добавлено

- маркер версии внутри распространяемого шаблона;
- SHA-256 для каждого выпускного ZIP;
- сквозная проверка состава, безопасности, контрольной суммы и развёртывания архива;
- автоматическая сборка артефакта и GitHub Release для тегов `v*`.

### Изменено

- файлы ZIP сортируются, а их внутренние временные метки фиксируются, поэтому одинаковые исходники дают одинаковый SHA-256;
- запись ZIP и SHA-256 выполняется через временные файлы, чтобы не оставлять частичный выпуск;
- версия, changelog, содержимое шаблона и Git-тег проверяются на согласованность.

## 0.1.1 — 2026-07-15

### Исправлено

- исправлена инструкция, ссылавшаяся на несуществующую папку `starter-kit`;
- дата создания единой книги привязана к дате инициализации проекта;

- запрещён выход ссылок, источников и выходного файла манифеста за пределы проекта;
- временные каталоги тестов сделаны уникальными для параллельных запусков;
- перезапись выпускного ZIP теперь требует явного параметра `-Force`.

### Добавлено

- лицензия MIT для исходного кода, документации и распространяемого шаблона;
- негативные тесты опасных путей, неверных дат, дублирующихся ID и устаревшей единой книги;
- кроссплатформенная проверка на Ubuntu и Windows;
- единые правила окончаний строк через [`.gitattributes`](.gitattributes).

## 0.1.0 — 2026-07-15

- опубликовано переносимое ядро проектной базы знаний;
