# Проектная основа

**Project Control Plane для сложных проектов: управляемая база знаний, замкнутый контур контроля и безопасная работа руководителя с ИИ-оператором.**

Текущая версия шаблона: **0.13.0**.

«Проектная основа» превращает репозиторий в долговременную память и контур управления проектом. Она не заменяет Jira, GitHub Issues или другую систему выполнения работ: задачи остаются там, а репозиторий связывает стратегию, выгоды, контрольные точки, риски, доказательства, обзоры и решения. Агентная нейросеть работает как личный ассистент, технический писатель и оркестратор виртуальных специалистов, но существенные полномочия остаются у руководителя.

Для руководителя предусмотрен отдельный человеко-ориентированный слой: единый пульт, готовые формулировки задач и объяснения без обязательного знания Git, терминала и структуры файлов. Установка и обслуживание вынесены в техническую инструкцию.

## Что решает проект

- сохраняет цели, границы, текущее состояние и критерии успеха;
- связывает ожидаемые выгоды с метриками, владельцами и доказательствами результата;
- объединяет риски, проблемы, зависимости и изменения в одном контуре контроля;
- формирует краткий управленческий статус из канонических данных;
- проверяет просрочки, пропущенные настройки, допуски и состояние проекта;
- отделяет факты от расчётов, оценок, допущений и гипотез;
- ведёт историю решений и условий их пересмотра;
- превращает неизвестные данные в управляемый реестр вопросов;
- связывает утверждения с источниками и датами проверки;
- даёт нейросети постоянные инструкции и структуру работы с репозиторием;
- позволяет нескольким нейросетям работать параллельно без создания конкурирующих версий проекта;
- автоматически поддерживает локальные папки руководителя и участников и общий контекст ИИ на одной опубликованной редакции `main`;
- позволяет ИИ выполнять роли стратега, аналитика, архитектора, руководителя поставки, критика и других специалистов;
- требует сохранять результат в проектных файлах, а не только в чате;
- преобразует вложения в адресуемый Markdown и включает только нужные фрагменты;
- собирает единую книгу проекта и автоматически проверяет целостность.

## Как работает система

1. Руководитель задаёт ценность, ограничения, полномочия и допустимые отклонения.
2. Команда ведёт задачи во внешней рабочей системе, а в репозитории хранит цели, результаты, условия и доказательства.
3. ИИ читает конфигурацию, статус, [`AGENTS.md`](./template/AGENTS.md), решения, вопросы и источники.
4. ИИ выбирает виртуальных специалистов, проводит анализ и обновляет канонические файлы.
5. Статус и здоровье проекта проверяются автоматически; отклонения превращаются в эскалации или решения.
6. Руководитель утверждает существенные решения, принятие риска и публикацию.
7. Obsidian используется для чтения и навигации, GitHub — для истории и проверки изменений.

## Для каких проектов подходит

- разработка продуктов и программно-аппаратных систем;
- корпоративные и государственные инициативы;
- исследовательские и инновационные проекты;
- проекты с несколькими рабочими потоками и участниками;
- проекты с правовыми, финансовыми или отраслевыми ограничениями;
- длительная работа руководителя с агентными нейросетями.

## Быстрый старт нового проекта

В Windows первоначальную настройку можно запустить двойным щелчком по [`template/START-PROJECT.cmd`](./template/START-PROJECT.cmd). В официальном выпускном архиве откроется встроенное визуальное окно Electron: мастер проведёт по пяти шагам, позволит выбрать несколько нейросетей и Obsidian, проверит их наличие, выполнит предварительную проверку без записи и изменит файлы только после явного подтверждения. Руководитель затем начинает с [`template/HOME.md`](./template/HOME.md) и [`template/START-HERE.md`](./template/START-HERE.md).

Для технической настройки требуется PowerShell 7 (`pwsh`). Для автоматической обработки бинарных документов дополнительно требуется Python 3.10+ и Microsoft MarkItDown.

1. Скопируйте содержимое папки `template` в новый репозиторий.
2. Запустите мастер:

       pwsh ./scripts/setup-project.ps1

3. Ответьте на вопросы, выберите нужные нейросети и Obsidian, затем подтвердите показанный план. Мастер заполнит [`PROJECT-CONFIG.json`](./template/PROJECT-CONFIG.json) и [`AI-TOOLS.json`](./template/AI-TOOLS.json), выполнит проверки и при доступном GitHub автоматически сделает проверку единой версии обязательной для `main`.
4. Добавьте первую измеримую выгоду в [`OUTCOMES.md`](./template/OUTCOMES.md) и ссылку на ближайшую работу.
5. Подключите агентную нейросеть по [`AI-CONNECTIONS.md`](./template/AI-CONNECTIONS.md).
6. При необходимости откройте проект в Obsidian и закрепите [`HOME.md`](./template/HOME.md).
7. Постройте статус и проверьте здоровье проекта:

       pwsh ./scripts/build-status.ps1
       pwsh ./scripts/check-project-health.ps1

## Пример задачи руководителя

> Подготовь проект к выбору варианта реализации. Изучи актуальные сведения, сравни варианты и дай рекомендацию. Сам обнови проект и выполни проверки. Не меняй цель, границы, бюджет и обязательства без моего подтверждения. В конце покажи риски и решения, которые нужны от меня.

## Контекст и вложения

Локальный пакет без вложений:

    pwsh ./scripts/build-context.ps1 -Profile compact -IncludeId D-001,Q-001 -Check
    pwsh ./scripts/check-context-health.ps1 -UpdateBaseline -Check

PDF, DOCX, PPTX, XLSX и другие файлы хранятся в `_attachments`, регистрируются в [`SOURCES.md`](./template/SOURCES.md) и локально преобразуются Microsoft MarkItDown:

    python -m pip install "markitdown[pdf,docx,pptx,xlsx,xls,outlook]"
    pwsh ./scripts/ingest-sources.ps1 -SourceId S-014
    pwsh ./scripts/build-ai-package.ps1 -Profile standard -ExpandSource S-014 -SourceQuery "сроки, ответственность" -RefreshSources -Check

Скрипты импорта работают только с локальными файлами, не загружают URL и не вызывают облачный OCR.

## Обновление существующего проекта

Распакуйте новый выпуск отдельно и сначала постройте план:

    pwsh ./scripts/update-project.ps1 -ProjectPath "../my-project"

После проверки примените его с `-Apply`. Обновление сохраняет канонические материалы, сверяет управляемые файлы с историческими SHA-256, создаёт резервную копию и откатывается при ошибке.

## Проверка шаблона

    pwsh ./scripts/test-template.ps1
    pwsh ./scripts/test-project-control.ps1
    pwsh ./scripts/test-ingestion.ps1
    pwsh ./scripts/test-agent-guides.ps1
    pwsh ./scripts/test-human-first.ps1
    pwsh ./scripts/test-setup-wizard.ps1
    pwsh ./scripts/test-release.ps1

Проверки выполняются на Ubuntu и Windows, проверяют структуру, ссылки, версии, миграции, управленческий цикл, контекст, вложения и наличие инструкций для агентных инструментов.

## Выпуск

    pwsh ./scripts/export-template.ps1

Готовый архив и SHA-256 появляются в `dist`. Проверка pull request заранее запрещает повторно использовать уже выпущенный номер. После слияния в `main` проверка сверяет все файлы версии, встраивает Windows-runtime визуального мастера, собирает архив, автоматически создаёт недостающий тег вида `v0.13.0` и GitHub Release. Существующий тег никогда не переписывается.

## Принципы

1. Репозиторий является источником состояния; чат — временным интерфейсом.
2. Руководитель сохраняет полномочия по целям, границам, бюджету, обязательствам и принятию рисков.
3. ИИ самостоятельно выполняет рутинную проектную и документальную работу.
4. История решений сохраняется и не переписывается задним числом.
5. Факты отделяются от оценок и подтверждаются источниками.
6. Существенные изменения проверяются через Git diff и pull request.
7. Методика остаётся переносимой и не требует одного поставщика моделей или редактора.
