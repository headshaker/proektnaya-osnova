# Проектная основа

**Каркас управляемой базы знаний для разработки сложных проектов с участием человека и нейросетей.**

Текущая версия шаблона: **0.1.1**.

«Проектная основа» помогает вести продуктовые, технологические, организационные и регулируемые проекты как единую проверяемую систему знаний, а не как набор переписок и разрозненных документов.

## Что решает проект

- сохраняет исходные условия, границы и текущую стадию проекта;
- отделяет факты от расчётов, оценок, допущений и гипотез;
- ведёт неизменяемую историю решений и условий их пересмотра;
- превращает неизвестные данные в управляемый реестр вопросов;
- связывает внешние утверждения с источниками и датами проверки;
- передаёт контекст между людьми и нейросетями без восстановления истории чата;
- собирает единую книгу проекта из канонических документов;
- автоматически проверяет метаданные, ссылки, идентификаторы и воспроизводимость выпуска.

## Для каких проектов подходит

- разработка новых продуктов и программно-аппаратных систем;
- корпоративные и государственные инициативы;
- исследовательские и инновационные проекты;
- проекты с несколькими рабочими потоками и участниками;
- проекты с правовыми, финансовыми или отраслевыми ограничениями;
- длительная совместная работа команды с нейросетями.

## Состав репозитория

| Путь | Назначение |
|---|---|
| `template/` | готовый каркас нового проекта |
| `scripts/test-template.ps1` | сквозная проверка развёртывания шаблона |
| `scripts/export-template.ps1` | выпуск отдельного ZIP с содержимым шаблона |
| `REPOSITORY.md` | рекомендуемые настройки GitHub |
| `CONTRIBUTING.md` | правила развития проекта |
| `ROADMAP.md` | направления следующих версий |

## Быстрый старт нового проекта

Требуется PowerShell 7 (`pwsh`). Скрипты проверяются на актуальных образах Ubuntu и Windows.

1. Скопируйте содержимое папки `template` в новый репозиторий.
2. Выполните:

       pwsh ./scripts/init-project.ps1 -Title "Название проекта" -Slug "project-slug"

3. Заполните `PROJECT-BRIEF.md`.
4. Утвердите первый контрольный рубеж и владельцев вопросов P0.
5. Работайте по протоколу из `AGENTS.md`.

## Проверка шаблона

Из корня репозитория «Проектная основа»:

    pwsh ./scripts/test-template.ps1

Проверка создаёт чистую копию, инициализирует тестовый проект, собирает единую книгу, проверяет ссылки и удаляет временные файлы.

## Выпуск разворачиваемого архива

    pwsh ./scripts/export-template.ps1

Готовый архив появится в `dist/proektnaya-osnova-template.zip`.
Если файл уже существует, явно разрешите замену:

    pwsh ./scripts/export-template.ps1 -Force

## Принципы

1. Канонические документы важнее производных представлений.
2. История решений сохраняется, а не переписывается задним числом.
3. Неизвестное фиксируется явно.
4. Автоматические проверки работают без обязательного сетевого доступа.
5. Нейросеть помогает вести проект, но не подменяет владельца решения.
6. Методика остаётся переносимой и не требует обязательной платформы или платного сервиса.

Проект находится на ранней стадии развития. Репозиторий уже публичен, но лицензия ещё не выбрана; до лицензионного решения права на использование и распространение явно не предоставлены.
