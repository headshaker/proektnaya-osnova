# Проектная основа

**Каркас управляемой базы знаний для развития сложных проектов руководителем и ИИ-оператором.**

Текущая версия шаблона: **0.6.0**.

«Проектная основа» превращает репозиторий в долговременную память и рабочую систему проекта. Руководитель формулирует цели, ограничения и решения, а агентная нейросеть работает как личный ассистент, технический писатель и оркестратор виртуальных специалистов: самостоятельно изучает состояние, изменяет канонические файлы, выполняет проверки и готовит изменения через Git.

## Что решает проект

- сохраняет цели, границы, текущее состояние и критерии успеха;
- отделяет факты от расчётов, оценок, допущений и гипотез;
- ведёт историю решений и условий их пересмотра;
- превращает неизвестные данные в управляемый реестр вопросов;
- связывает утверждения с источниками и датами проверки;
- даёт нейросети постоянные инструкции и структуру работы с репозиторием;
- позволяет ИИ выполнять роли стратега, аналитика, архитектора, руководителя поставки, критика и других специалистов;
- требует сохранять результат в проектных файлах, а не только в чате;
- преобразует вложения в адресуемый Markdown и включает только нужные фрагменты;
- собирает единую книгу проекта и автоматически проверяет целостность.

## Как работает система

1. Руководитель ставит задачу естественным языком.
2. ИИ читает [`AGENTS.md`](./template/AGENTS.md), состояние проекта, решения, вопросы и источники.
3. ИИ выбирает виртуальных специалистов и проводит профессиональный анализ.
4. ИИ самостоятельно изменяет нужные файлы и обновляет [`HANDOFF.md`](./template/HANDOFF.md).
5. ИИ запускает проверки и показывает управленческий итог.
6. Руководитель утверждает существенные решения и публикацию.
7. Obsidian используется для удобного чтения и навигации, GitHub — для истории и проверки изменений.

## Для каких проектов подходит

- разработка продуктов и программно-аппаратных систем;
- корпоративные и государственные инициативы;
- исследовательские и инновационные проекты;
- проекты с несколькими рабочими потоками и участниками;
- проекты с правовыми, финансовыми или отраслевыми ограничениями;
- длительная работа руководителя с агентными нейросетями.

## Быстрый старт нового проекта

Требуется PowerShell 7 (`pwsh`). Для автоматической обработки бинарных документов дополнительно требуется Python 3.10+ и Microsoft MarkItDown.

1. Скопируйте содержимое папки `template` в новый репозиторий.
2. Выполните:

       pwsh ./scripts/init-project.ps1 -Title "Название проекта" -Slug "project-slug"

3. Подключите агентную нейросеть по [`AI-CONNECTIONS.md`](./template/AI-CONNECTIONS.md).
4. Откройте проект как vault в Obsidian.
5. Передайте агенту первую задачу из [`START-HERE.md`](./template/START-HERE.md).

## Пример задачи руководителя

> Подготовь проект к выбору варианта реализации. Сам изучи репозиторий, выбери нужных виртуальных специалистов, сравни варианты, обнови канонические файлы и [`HANDOFF.md`](./template/HANDOFF.md), добавь недостающие вопросы и запусти проверки. Не меняй цель, границы, бюджет и обязательства без моего подтверждения. В конце покажи рекомендацию, риски, изменённые файлы и решения, которые нужны от меня.

## Контекст и вложения

Локальный пакет без вложений:

    pwsh ./scripts/build-context.ps1 -Profile compact -IncludeId D-001,Q-001 -Check

PDF, DOCX, PPTX, XLSX и другие файлы хранятся в `_attachments`, регистрируются в [`SOURCES.md`](./template/SOURCES.md) и локально преобразуются Microsoft MarkItDown:

    python -m pip install "markitdown[pdf,docx,pptx,xlsx,xls,outlook]"
    pwsh ./scripts/ingest-sources.ps1 -SourceId S-014
    pwsh ./scripts/build-ai-package.ps1 -Profile standard -ExpandSource S-014 -SourceQuery "сроки, ответственность" -RefreshSources -Check

Скрипты импорта работают только с локальными файлами, не загружают URL и не вызывают облачный OCR.

## Обновление существующего проекта

Распакуйте новый выпуск отдельно и сначала постройте план:

    pwsh ./scripts/update-project.ps1 -ProjectPath "../my-project"

После проверки примените его с `-Apply`. Обновление сохраняет канонические материалы, сверяет управляемые файлы с историческими SHA-256, создаёт резервную копию и откатывается при ошибке.

## Проверка шаблона

    pwsh ./scripts/test-template.ps1
    pwsh ./scripts/test-ingestion.ps1
    pwsh ./scripts/test-agent-guides.ps1
    pwsh ./scripts/test-release.ps1

Проверки выполняются на Ubuntu и Windows, проверяют структуру, ссылки, версии, миграции, контекст, вложения и наличие инструкций для агентных инструментов.

## Выпуск

    pwsh ./scripts/export-template.ps1

Готовый архив и SHA-256 появляются в `dist`. Тег вида `v0.6.0`, совпадающий с [`VERSION`](./VERSION), автоматически создаёт GitHub Release.

## Принципы

1. Репозиторий является источником состояния; чат — временным интерфейсом.
2. Руководитель сохраняет полномочия по целям, границам, бюджету, обязательствам и принятию рисков.
3. ИИ самостоятельно выполняет рутинную проектную и документальную работу.
4. История решений сохраняется и не переписывается задним числом.
5. Факты отделяются от оценок и подтверждаются источниками.
6. Существенные изменения проверяются через Git diff и pull request.
7. Методика остаётся переносимой и не требует одного поставщика моделей или редактора.
