## Рекомендуемые настройки

1. Включить Issues и Discussions после появления первых внешних пользователей.
2. Запретить прямую запись в `main`; изменения принимать через pull request.
3. Сделать проверку `Проверка репозитория шаблона` обязательной.
4. Требовать хотя бы одно подтверждение для изменений протокола, реестров и автоматических проверок.
5. Включить автоматическое удаление веток после слияния.
6. Не включать режим GitHub template repository, пока каркас хранится в подпапке `template`; для новых проектов использовать выпускной ZIP либо содержимое этой папки.
